# SEC 美股公告监控系统

## 一、项目目标

开发一套基于 Golang 的 SEC 公告监控系统，支持用户通过 Web 页面配置需要关注的美股股票及 ETF 标的，系统自动从 SEC 官方数据源周期性拉取相关公告信息，并提供统一的查询、筛选、审计、通知和配置管理能力。

---

# 二、功能需求

## 2.1 标的管理

系统提供标的管理页面，支持：

* 新增监控标的
* 编辑监控标的
* 删除监控标的
* 启用/禁用监控标的

每个标的至少包含：

* 股票代码（Ticker）
* 公司名称
* CIK（如可获取）
* 标的类型

  * Stock（股票）
  * ETF（基金）
* 创建时间
* 更新时间
* 状态

支持同时配置多个监控标的。

---

## 2.2 SEC 公告拉取

系统启动后，根据已配置标的自动执行公告拉取任务。

数据来源：

* SEC EDGAR 官方接口

支持监控以下公告类型（后续可扩展）：

* 8-K
* 10-K
* 10-Q
* S-1
* S-3
* Form 4
* 13F
* 其他 SEC Filing

系统需记录：

* Filing ID
* Filing Type
* Ticker
* Company Name
* Filing Date
* Filing URL
* Filing Title
* 原始数据内容（可选）
* 拉取时间

要求：

* 避免重复入库
* 支持增量同步
* 支持失败重试
* 支持日志记录

---

## 2.3 拉取任务调度

系统提供调度配置页面。

支持：

* Cron 表达式配置
* 手动立即执行
* 启用/禁用任务

示例：

* 每5分钟执行一次
* 每30分钟执行一次
* 每小时执行一次

配置修改后应尽量动态生效，无需重启服务。

---

## 2.4 公告查询页面

提供公告列表页面。

支持：

### 查询条件

* 股票代码
* 公司名称
* Filing 类型
* 时间范围

### 列表展示

显示：

* Filing Type
* Ticker
* Company Name
* Filing Date
* 发布时间
* SEC 官方链接

### 操作

* 查看详情
* 跳转 SEC 官方页面
* 刷新数据

### 排序

默认：

* Filing Date DESC

---

## 2.5 Telegram 通知

系统提供 Telegram 通知配置页面。

支持：

### Telegram 配置

* Bot Token
* Chat ID
* 启用/禁用通知

### 通知规则

当开启 Telegram 通知后：

* 系统每次拉取到新的 SEC 公告时自动发送通知
* 仅对新增公告发送通知，避免重复推送
* 通知内容包含：

  * 股票代码（Ticker）
  * 公司名称
  * Filing 类型
  * Filing 标题
  * Filing 日期
  * SEC 官方链接

### 通知要求

* 支持通知失败重试
* 支持记录通知日志
* 支持测试发送功能
* 配置修改后动态生效

---

## 2.6 操作审计日志

系统记录所有关键操作。

记录内容：

* 操作时间
* 操作用户（预留）
* 操作类型
* 操作对象
* 操作前内容
* 操作后内容

需要记录：

* 新增标的
* 修改标的
* 删除标的
* 修改调度配置
* 修改系统配置
* 修改 Telegram 配置

提供独立审计日志页面。

---

## 2.7 分页与刷新机制

所有表格页面统一支持：

### 分页

参数：

* Page
* PageSize

支持：

* 首页
* 上一页
* 下一页
* 跳页

### 排序

默认：

* 时间倒序

### 刷新

支持：

* 手动刷新按钮
* 自动刷新

自动刷新间隔可配置。

---

## 2.8 系统配置中心

提供独立配置管理页面。

配置项包括：

### SEC配置

* SEC API地址
* User-Agent
* 请求超时

### 调度配置

* Cron表达式
* 自动刷新间隔

### Telegram配置

* Bot Token
* Chat ID
* 是否启用通知

### 数据库配置

* 数据库类型
* 数据库路径

### 系统配置

* 日志级别
* 数据保留天数

要求：

* 配置持久化保存
* 动态加载
* 配置变更即时生效（尽可能）
